# プロジェクト概要
目的: 解答の最終結果ではなく、試行錯誤の「プロセス（筆記・停止・消去）」をAIで評価・称賛するデジタルノートアプリ（動くデモ用プロトタイプ）。
ターゲット環境: iPad + Apple Pencil (Safari等のWebブラウザで動作するPWA想定)

# 技術スタック
* **フロントエンド:** Next.js (React), TypeScript
* **描画ライブラリ:** Perfect-Freehand または Fabric.js (Pointer Eventsを用いた筆圧検知対応)
* **バックエンド:** Python (FastAPI)
* **AI:** Google Gemini API (マルチモーダル機能の活用)

# 1. フロントエンド実装要件 (Next.js)
## 1.1 UIレイアウト
* **キャンバスエリア:** 問題画像（固定）を背景にし、その上に手書き用の透明なCanvasレイヤーを配置。
* **コントロールエリア:** 「ペン / 消しゴム」の切り替え、分析実行（送信）ボタン、リプレイ（再生）ボタン。
* **フィードバックエリア:** バックエンドから返ってきたAIの評価（JSON）を表示するサイドパネル。

## 1.2 コア機能とデータ構造
* **ストロークのロギング:** 描画と消去の軌跡を時間情報とともにJSON配列として記録する。
    * データ構造イメージ:
        `[{ "strokeId": "s_01", "type": "draw"|"erase", "startTime": 1678886400, "endTime": 1678886405, "points": [{"x": 10, "y": 20, "p": 0.5, "t": 10}, ...] }]`
* **リプレイ（タイムラプス）機能:** 記録したストローク配列のタイムスタンプを使用し、キャンバス上に実際の描画過程をアニメーションで再描画する機能。

# 2. バックエンド実装要件 (FastAPI + Python)
## 2.1 ログの画像化処理（Ghost Rendering手法）
* フロントエンドからストロークのJSONデータを受信し、サーバー側で1枚の画像にレンダリングする。
* **描画ルール:** 現在キャンバスに残っている線（draw）は「黒色」、一度書かれてから消しゴムで消された線（erase）は「半透明の赤色」で描画する。（これにより消された思考プロセスを可視化する）

## 2.2 停止時間（迷い）の計算
* ストローク間のタイムスタンプの差分（Δt）を計算し、一定時間（例: 10秒以上）の空白があれば「難所での思考時間」として検知し、メタデータに付与する。

# 3. AI連携実装要件 (Gemini API)
## 3.1 プロンプト設計
* 2.1で生成した「黒色＋半透明の赤色の線が含まれた画像」と、問題の正解方針（テキスト）、2.2の「迷い時間」のデータをGemini API（Pro または Flash）にマルチモーダルで送信。
* **AIへの指示内容:**
    1. 半透明の赤色の線は「一度書いて消した仮説（試行錯誤の痕跡）」である。
    2. 最終的な計算結果（黒色）が間違っていても、赤色・黒色のプロセスの中に「正しい方針」が含まれていれば、それを抽出して強く称賛すること。
    3. 消去履歴から視点の切り替え（柔軟性）を評価すること。
* **出力形式:** JSON形式で以下を返すように指定。
    `{ "総合評価": string, "プロセスへの称賛ポイント": [string], "惜しい点（ヒント）": string, "思考タイプラベル": string }`
